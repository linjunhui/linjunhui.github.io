## 快速开始

### 1. 创建仓库

1. 注册 GitHub 账号（如果还没有）并确认邮箱（必需！）
2. 点击右上角的 "Use this template" 按钮
3. 在 "New repository" 页面，输入仓库名称为 `[你的GitHub用户名].github.io`，这将是您网站的 URL
4. 配置网站设置并添加内容
5. 上传文件（如 PDF、.zip 等）到 `files/` 目录，它们将出现在 `https://[你的GitHub用户名].github.io/files/example.pdf`
6. 在仓库设置的 "GitHub pages" 部分检查状态

### 2. 网站地址

您的网站地址将是：`https://[你的GitHub用户名].github.io`

## 网站配置

### 主要配置文件：`_config.yml`

这是网站的核心配置文件，位于项目根目录。主要配置项包括：

#### 基本站点设置

```yaml
title: "您的姓名 / 网站标题"
name: "您的姓名"
description: "您的学术作品集描述"
url: https://yourusername.github.io
baseurl: ""
```

#### 作者信息设置

在 `author:` 部分配置您的个人信息：

```yaml
author:
  avatar: "profile.png"  # 头像文件名，放在 images/ 目录
  name: "侧边栏显示的名称"
  bio: "左侧边栏的简短个人简介"
  location: "所在地"
  employer: "工作单位"
  email: "your.email@example.org"
  
  # 学术网站链接
  googlescholar: "https://scholar.google.com/citations?user=..."
  orcid: "https://orcid.org/yourorcidurl"
  
  # 社交媒体链接
  github: "yourusername"
  twitter: "yourusername"
  linkedin: "yourusername"
  # ... 更多社交媒体平台
```

#### 主题设置

```yaml
site_theme: "default"  # 可选: "default" 或 "air"
locale: "en-US"  # 语言设置
```

#### 其他重要设置

- `breadcrumbs`: 是否显示面包屑导航（true/false）
- `read_more`: 是否启用"阅读更多"链接
- `comments`: 评论系统配置（disqus、discourse 等）
- `analytics`: 分析工具配置（Google Analytics 等）

## 添加页面

### 页面类型

网站支持以下几种页面类型：

1. **单页** (`_pages/` 目录)
2. **博客文章** (`_posts/` 目录)
3. **出版物** (`_publications/` 目录)
4. **演讲** (`_talks/` 目录)
5. **教学** (`_teaching/` 目录)
6. **作品集** (`_portfolio/` 目录)

### 创建单页

在 `_pages/` 目录下创建 `.md` 或 `.html` 文件：

**示例：创建"关于"页面**

```markdown
---
permalink: /about/
title: "关于我"
author_profile: true
---

这是关于页面的内容，可以使用 Markdown 格式编写。
```

**重要字段说明：**

- `permalink`: 页面的 URL 路径
- `title`: 页面标题
- `author_profile`: 是否显示作者侧边栏（true/false）
- `layout`: 布局类型（single、archive 等）

### 创建博客文章

在 `_posts/` 目录下创建文件，文件名格式：`YYYY-MM-DD-文章标题.md`

```markdown
---
title: "我的第一篇博客文章"
date: 2024-01-01
categories:
  - 分类1
tags:
  - 标签1
  - 标签2
---

文章内容...
```

### 创建出版物

在 `_publications/` 目录下创建文件：

```markdown
---
title: "论文标题"
collection: publications
category: manuscripts  # 或 conferences, books
permalink: /publication/2024-01-01-paper-title
excerpt: '论文摘要'
date: 2024-01-01
venue: '期刊名称'
paperurl: 'https://example.com/paper.pdf'
citation: '作者名. (2024). "论文标题." <i>期刊名</i>. 1(1).'
---

论文详细描述...
```

**分类选项：**
- `manuscripts`: 期刊文章
- `conferences`: 会议论文
- `books`: 书籍

### 创建演讲

在 `_talks/` 目录下创建文件：

```markdown
---
title: "演讲标题"
collection: talks
type: "Talk"  # 或 "Tutorial", "Poster"
permalink: /talks/2024-01-01-talk-title
venue: "会议或机构名称"
date: 2024-01-01
location: "城市, 国家"
---

演讲描述...
```

### 创建教学页面

在 `_teaching/` 目录下创建文件：

```markdown
---
title: "课程名称"
collection: teaching
permalink: /teaching/2024-spring-course-name
venue: "大学名称"
date: 2024-01-01
---

课程描述...
```

### 创建作品集

在 `_portfolio/` 目录下创建文件：

```markdown
---
title: "项目名称"
collection: portfolio
permalink: /portfolio/project-name
excerpt: '项目简介'
---

项目详细描述...
```

## 添加内容

### 文件结构说明

```
项目根目录/
├── _config.yml          # 网站配置文件
├── _data/
│   └── navigation.yml   # 导航菜单配置
├── _pages/              # 单页文件
├── _posts/              # 博客文章
├── _publications/       # 出版物
├── _talks/              # 演讲
├── _teaching/           # 教学
├── _portfolio/          # 作品集
├── files/               # 静态文件（PDF等）
└── images/              # 图片文件
```

### Markdown 语法

网站使用 Jekyll Kramdown 解析器，支持 GitHub Flavored Markdown (GFM)。

**常用语法：**

- **标题**: `# 一级标题`, `## 二级标题`
- **粗体**: `**粗体文本**`
- **斜体**: `*斜体文本*`
- **链接**: `[链接文本](URL)`
- **图片**: `![alt文本](图片路径)`
- **代码**: `` `代码` `` 或 ```代码块```
- **列表**: `- 项目` 或 `1. 项目`
- **引用**: `> 引用内容`

### 数学公式支持

使用 MathJax 支持数学公式：

- 行内公式: `\(公式\)` 或 `$公式$`
- 块级公式: `$$公式$$` 或 `\[公式\]`

### 图表支持

- **Mermaid 图表**: 使用代码块语法
- **Plotly 图表**: 使用 JSON 格式的代码块

详细语法请参考 `/markdown/` 页面。

## 导航菜单设置

### 配置文件：`_data/navigation.yml`

编辑此文件来修改顶部导航菜单：

```yaml
main:
  - title: "出版物"
    url: /publications/
  
  - title: "演讲"
    url: /talks/
  
  - title: "教学"
    url: /teaching/
  
  - title: "作品集"
    url: /portfolio/
  
  - title: "博客"
    url: /year-archive/
  
  - title: "简历"
    url: /cv/
  
  - title: "指南"
    url: /markdown/
```

**说明：**
- 修改 `title` 来更改菜单显示文本
- 修改 `url` 来更改链接地址
- 删除某项可以隐藏该菜单项
- 添加新项可以添加新菜单

## 本地运行

### 使用 Ruby 和 Bundler

1. **安装依赖**

   Linux/WSL:
   ```bash
   sudo apt install ruby-dev ruby-bundler nodejs
   ```

   macOS:
   ```bash
   brew install ruby node
   gem install bundler
   ```

2. **安装项目依赖**
   ```bash
   bundle install
   ```

   如果遇到权限错误，可以设置本地安装路径：
   ```bash
   bundle config set --local path 'vendor/bundle'
   bundle install
   ```

3. **启动本地服务器**
   ```bash
   bundle exec jekyll serve -l -H localhost
   ```

   访问 `http://localhost:4000` 查看网站

### 使用 Docker

```bash
chmod -R 777 .
docker compose up
```

访问 `http://localhost:4000`

### 使用 VS Code DevContainer

1. 在 VS Code 中打开项目
2. 按 `F1` -> 选择 `DevContainer: Reopen in Container`
3. 网站将自动在 `http://localhost:4000` 运行

## 常见问题

### 1. 如何修改网站标题和描述？

编辑 `_config.yml` 文件中的 `title` 和 `description` 字段。

### 2. 如何添加新的导航菜单项？

编辑 `_data/navigation.yml` 文件，添加新的菜单项。

### 3. 如何上传文件（如 PDF）？

将文件上传到 `files/` 目录，然后通过 `/files/文件名.pdf` 访问。

### 4. 如何修改头像？

1. 将头像图片放到 `images/` 目录
2. 在 `_config.yml` 的 `author.avatar` 字段中指定文件名

### 5. 如何隐藏某个页面在导航菜单中？

从 `_data/navigation.yml` 中删除对应的菜单项。页面仍然可以通过直接 URL 访问。

### 6. 如何添加评论功能？

在 `_config.yml` 中配置 `comments` 部分，支持 Disqus、Discourse 等。

### 7. 如何添加 Google Analytics？

在 `_config.yml` 中配置 `analytics` 部分：
```yaml
analytics:
  provider: "google-analytics-4"
  google:
    tracking_id: "你的跟踪ID"
```

### 8. 如何修改主题？

在 `_config.yml` 中修改 `site_theme` 字段：
- `default`: 默认主题
- `air`: 简洁主题

### 9. 页面更新后没有显示？

- 检查文件名和 front matter 格式是否正确
- 确认文件在正确的目录下
- 本地运行 `jekyll serve` 检查错误信息
- 查看 GitHub Actions 的构建状态

### 10. 如何批量生成内容？

使用 `markdown_generator/` 目录下的 Jupyter notebooks 或 Python 脚本，可以从 CSV/TSV 文件批量生成 Markdown 文件。

## 提示

1. **文件命名**: 使用小写字母和连字符，避免空格和特殊字符
2. **图片优化**: 压缩图片以提高加载速度
3. **定期备份**: 定期提交更改到 Git 仓库
4. **测试本地**: 在推送到 GitHub 之前，先在本地测试更改
5. **查看日志**: 如果遇到问题，查看 Jekyll 构建日志或 GitHub Actions 日志
