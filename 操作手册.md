## 快速开始

### 1. 首次设置

1. **修改网站基本信息**
   - 编辑 `_config.yml` 文件
   - 修改 `title`、`name`、`description`、`url` 等基本设置
   - 更新 `author` 部分的个人信息

2. **设置头像**
   - 将头像图片放到 `images/` 目录
   - 在 `_config.yml` 的 `author.avatar` 中指定文件名

3. **配置导航菜单**
   - 编辑 `_data/navigation.yml`
   - 添加或删除菜单项

### 2. 部署网站

1. 提交更改到 GitHub
2. 等待 GitHub Pages 自动构建（通常几分钟）
3. 访问 `https://[你的用户名].github.io` 查看网站

## 网站基础配置

### 修改 `_config.yml`

#### 基本站点信息

```yaml
title: "你的网站标题"
name: "你的姓名"
description: "网站描述"
url: https://yourusername.github.io
```

#### 作者信息

```yaml
author:
  avatar: "profile.png"  # 头像文件名
  name: "你的姓名"
  bio: "个人简介"
  location: "所在地"
  email: "your.email@example.com"
  github: "yourusername"
  # ... 其他社交媒体链接
```

#### 主题设置

```yaml
site_theme: "default"  # 或 "air"
locale: "zh-CN"  # 中文设置
```

#### 时区设置

```yaml
timezone: Asia/Shanghai  # 中国时区
```

## 添加博客文章

### 1. 创建文章文件

在 `_posts/` 目录下创建文件，文件名格式：
```
YYYY-MM-DD-文章标题.md
```

示例：`2024-01-15-my-first-post.md`

### 2. 文章 Front Matter

```yaml
---
title: "文章标题"
date: 2024-01-15
permalink: /posts/2024/01/my-first-post/
categories:
  - 分类1
  - 分类2
tags:
  - 标签1
  - 标签2
excerpt: "文章摘要，会显示在列表页"
---
```

### 3. 文章内容

使用 Markdown 格式编写内容：

```markdown
## 标题

正文内容...

### 子标题

- 列表项1
- 列表项2

**粗体** *斜体*

[链接文本](URL)

![图片描述](图片路径)
```

### 4. 完整示例

```markdown
---
title: "我的第一篇博客文章"
date: 2024-01-15
permalink: /posts/2024/01/my-first-post/
categories:
  - 技术
tags:
  - Python
  - 教程
excerpt: "这是一篇关于 Python 的文章..."
---

## 引言

这是文章的开头...

## 主要内容

### 第一部分

内容...

## 总结

文章总结...
```

## 创建合集（系列文章）

### 1. 创建系列文章

在文章的 front matter 中添加：

```yaml
---
title: "SGLang 源码学习 0 - 架构分析"
date: 2024-01-15
categories:
  - SGLang 源码学习
tags:
  - SGLang
series: "SGLang 源码学习"  # 系列名称（必须）
series_order: 0  # 顺序号（必须）
---
```

### 2. 系列文章命名规范

- **文件名**: `YYYY-MM-DD-series-name-part-number.md`
- **标题**: 建议包含系列名称和序号
- **顺序号**: 从 0 开始，使用数字（0, 1, 2, ...）

### 3. 示例

**第 0 篇**:
```yaml
---
title: "SGLang 源码学习 0 - SGLang 架构分析"
series: "SGLang 源码学习"
series_order: 0
---
```

**第 1 篇**:
```yaml
---
title: "SGLang 源码学习 1 - Tokenizer 实现分析"
series: "SGLang 源码学习"
series_order: 1
---
```

### 4. 系列导航功能

- 自动显示系列文章列表
- 高亮当前文章
- 显示上一篇/下一篇链接
- 显示系列文章总数

**注意**: 系列导航会自动出现在文章顶部，无需手动添加。

### 5. 在中间插入文章

如果需要在一系列文章中间插入新文章，可以使用小数：

```yaml
series_order: 0.5  # 插入在第 0 和第 1 篇之间
```

## 添加出版物

### 1. 创建出版物文件

在 `_publications/` 目录下创建文件，文件名格式：
```
YYYY-MM-DD-paper-title.md
```

### 2. Front Matter 格式

```yaml
---
title: "论文标题"
collection: publications
category: manuscripts  # manuscripts, conferences, books
permalink: /publication/2024-01-15-paper-title
excerpt: '论文摘要'
date: 2024-01-15
venue: '期刊名称'
paperurl: 'https://example.com/paper.pdf'
citation: '作者名. (2024). "论文标题." <i>期刊名</i>. 1(1).'
---
```

### 3. 分类说明

- **`manuscripts`**: 期刊文章
- **`conferences`**: 会议论文
- **`books`**: 书籍

### 4. 完整示例

```yaml
---
title: "Deep Learning for NLP"
collection: publications
category: manuscripts
permalink: /publication/2024-01-15-deep-learning-nlp
excerpt: '本文提出了一种新的深度学习方法...'
date: 2024-01-15
venue: 'Journal of AI Research'
paperurl: 'https://example.com/paper.pdf'
citation: 'Zhang, L. (2024). "Deep Learning for NLP." <i>Journal of AI Research</i>. 1(1).'
---

论文详细描述...
```

## 添加单页

### 1. 创建单页文件

在 `_pages/` 目录下创建 `.md` 或 `.html` 文件

### 2. Front Matter 格式

```yaml
---
permalink: /about/
title: "关于我"
author_profile: true
---
```

### 3. 常用选项

- **`permalink`**: 页面 URL
- **`title`**: 页面标题
- **`author_profile`**: 是否显示作者侧边栏（true/false）
- **`layout`**: 布局类型（single, archive 等）

### 4. 完整示例

```yaml
---
permalink: /about/
title: "关于我"
author_profile: true
---

这是关于页面的内容...
```

## 管理导航菜单

### 1. 编辑导航文件

编辑 `_data/navigation.yml` 文件

### 2. 添加菜单项

```yaml
main:
  - title: "出版物"
    url: /publications/
  
  - title: "演讲"
    url: /talks/
  
  - title: "新菜单项"
    url: /new-page/
```

### 3. 删除菜单项

从 `navigation.yml` 中删除对应的项即可

### 4. 调整顺序

改变 `navigation.yml` 中的顺序即可调整菜单显示顺序

## 上传文件

### 1. 上传 PDF 等文件

1. 将文件放到 `files/` 目录
2. 文件会自动出现在 `https://your-site.github.io/files/文件名.pdf`

### 2. 上传图片

1. 将图片放到 `images/` 目录
2. 在文章中使用：`![描述](images/图片名.png)`

### 3. 在文章中引用文件

```markdown
[下载PDF](/files/paper.pdf)
[下载幻灯片](/files/slides.pdf)
```

## 本地运行和调试

### 1. 安装依赖

**Linux/WSL:**
```bash
sudo apt install ruby-dev ruby-bundler nodejs
```

**macOS:**
```bash
brew install ruby node
gem install bundler
```

### 2. 安装项目依赖

```bash
bundle install
```

如果遇到权限错误：
```bash
bundle config set --local path 'vendor/bundle'
bundle install
```

### 3. 启动本地服务器

```bash
bundle exec jekyll serve -l -H localhost
```

访问 `http://localhost:4000` 查看网站

### 4. 使用 Docker

```bash
chmod -R 777 .
docker compose up
```

### 5. 使用 VS Code DevContainer

1. 在 VS Code 中打开项目
2. 按 `F1` -> 选择 `DevContainer: Reopen in Container`
3. 网站自动在 `http://localhost:4000` 运行

## 常见问题

### Q1: 如何修改网站标题和描述？

**A:** 编辑 `_config.yml` 文件中的 `title` 和 `description` 字段。

### Q2: 如何添加新的导航菜单项？

**A:** 编辑 `_data/navigation.yml` 文件，添加新的菜单项。

### Q3: 如何上传文件（如 PDF）？

**A:** 将文件上传到 `files/` 目录，然后通过 `/files/文件名.pdf` 访问。

### Q4: 如何修改头像？

**A:** 
1. 将头像图片放到 `images/` 目录
2. 在 `_config.yml` 的 `author.avatar` 字段中指定文件名

### Q5: 如何隐藏某个页面在导航菜单中？

**A:** 从 `_data/navigation.yml` 中删除对应的菜单项。页面仍然可以通过直接 URL 访问。

### Q6: 如何添加评论功能？

**A:** 在 `_config.yml` 中配置 `comments` 部分：

```yaml
comments:
  provider: "disqus"  # 或其他提供商
  disqus:
    shortname: "your-disqus-shortname"
```

### Q7: 如何添加 Google Analytics？

**A:** 在 `_config.yml` 中配置：

```yaml
analytics:
  provider: "google-analytics-4"
  google:
    tracking_id: "G-XXXXXXXXXX"
```

### Q8: 如何修改主题？

**A:** 在 `_config.yml` 中修改：

```yaml
site_theme: "air"  # 或 "default"
```

### Q9: 页面更新后没有显示？

**A:** 
- 检查文件名和 front matter 格式是否正确
- 确认文件在正确的目录下
- 本地运行 `jekyll serve` 检查错误信息
- 查看 GitHub Actions 的构建状态

### Q10: 如何批量生成内容？

**A:** 使用 `markdown_generator/` 目录下的 Jupyter notebooks 或 Python 脚本，可以从 CSV/TSV 文件批量生成 Markdown 文件。

### Q11: 如何创建系列文章？

**A:** 在文章的 front matter 中添加：
```yaml
series: "系列名称"
series_order: 0
```
详细说明见 [创建合集指南.md](创建合集指南.md)

### Q12: 如何修改文章永久链接格式？

**A:** 在 `_config.yml` 中修改：

```yaml
permalink: /:categories/:title/  # 或其他格式
```

### Q13: 如何添加数学公式？

**A:** 使用 MathJax 语法：

- 行内公式: `\(公式\)` 或 `$公式$`
- 块级公式: `$$公式$$` 或 `\[公式\]`

### Q14: 如何添加代码高亮？

**A:** 使用代码块语法：

````markdown
```python
print('Hello World!')
```
````

### Q15: 如何设置文章摘要？

**A:** 在 front matter 中添加 `excerpt` 字段，或在文章中使用 `<!--more-->` 分隔符。

## 最佳实践

### 1. 文件命名

- 使用小写字母和连字符
- 避免空格和特殊字符
- 日期格式：`YYYY-MM-DD`

### 2. 内容组织

- 保持文章长度适中
- 使用清晰的标题层级
- 添加适当的分类和标签

### 3. 图片优化

- 压缩图片以提高加载速度
- 使用描述性的文件名
- 添加 alt 文本

### 4. 版本控制

- 定期提交更改到 Git
- 使用有意义的提交信息
- 在推送到 GitHub 前先在本地测试

### 5. 系列文章

- 规划好整个系列的结构
- 保持文章顺序一致
- 在文章间添加适当的链接
